1. Create An Account

Email:

A valid Email address is required for all Compliance Investigate accounts. The Email address is used to login to a user's account. The provided Email address is used for any Compliance Investigate notifications, such as password reset request.

Password:

Compliance Investigate uses the following requirements for all passwords:

  • Minimum password length is 8 characters
  • Passwords must include uppercase, lowercase, and special characters

First And Last Name:

Please provide your First and Last Name.

2. Audit Options

Start At This Page

Enter the Full Path Web Address of the page you would like to scan. This defines the scope of the Audit. Please note the URL you provide should not be a page that redirects to another URL.

Common Error Messages:

  • Start Page is invalid - A valid http:// or https:// URL needs to be entered.

Guideline

Select the compliance guideline that you would like to audit your page against. The available guidelines include:

  • Section 508
  • WCAG 2.1 A
  • WCAG 2.1 AA
  • WCAG 2.1 AAA
The table below lists the guidelines that will be used by the Audit:
Guideline Description
Section 508 Checks against the accessibility standards as set forth by the US Access Board. Section 508 of the Rehabilitation Act Amendments mandates that federal agencies must ensure that all electronic and information technology purchased, maintained, or developed meets the accessibility standards established by the U.S. Access Board. For more information go to: http://www.access-board.gov/ict
WCAG 2.1 A Published by the W3C, the Web Content Accessibility Guidelines (WCAG) 2.1 A standards are meant to be the baseline of success criteria for accessibility compliance. WCAG 2.1 covers a wide range of recommendations for making Web content more accessible. Following these guidelines will make content accessible to a wider range of people with disabilities, including blindness and low vision, deafness and hearing loss, learning disabilities, cognitive limitations, limited movement, speech disabilities, photosensitivity and combinations of these.
WCAG 2.1 AA The WCAG 2.1 AA standards are meant to include all of the level A standards as well as more stringent criteria and conformance requirements.
WCAG 2.1 AAA The WCAG 2 AAA standards are meant to include all the A, AA criteria as well as the highest level of conformance requirements.

Total Pages To Crawl

Limit the maximum number of pages returned per audit between 1-25 pages

Number Of Levels To Crawl

In Compliance Investigate, users can specify the number of levels that an audit will track down. An audit will include any pages referenced by the current page that is being assessed. A level of zero (0) will only audit the page defined in the "Start at this page" field. An audit that uses 1 or more levels will spider the domain of the specified site.

Note: Increasing the number of levels used will increase Audit completion times.

3. Audit Results

Audit And Guideline Selection

An audit must be selected in order to show data in the report. By default, the first Audit alphabetically is selected. You can change the Audit selections at any time and the report will update accordingly. The Guideline selection is defaulted to All. This means that all Guidelines tested for this Audit will be included in the report. However, you can also change these selections at any time and the report will update.

Summary Bar

The Summary Bar section of the report will update based on the Audit and Guideline selections. The Summary Bar consists of the following metrics.

  • Pages Scanned – The total number of pages scanned for the selected Audit.
  • Pages Impacted – The total number of pages that had at least one issue associated as a result of a test performed.
  • Total Issues – The total number of issues that did not Pass a test across the Audits and Guidelines selected.
  • Total Failed – the total number of issues that Failed a test across the Audits and Guidelines selected.
  • High Severity Failed – The total number of issues that Failed a test and were a High Severity issue based on the Audits and Guidelines selected.

Issues Found

The Issues Found table includes the list of all issues that did not Pass a test based on the Audit and Guidelines selected.

Issues Found ‘count’ (found in parenthesis after the heading) includes the total number of issues that did not Pass a test. This initial list displayed on the screen will be the top 20 issues in priority order. You will have to select See More option at the end of the list to see more issues.

Table Columns:

  • Issue - The Result of a particular test performed on a particular page.
  • Severity - The Severity of an issue is determined by a combination of priority level and highest page level.

  • Impact – The Impact is the Total Pages with at least one Issue / Total Pages Scanned.
  • Occurrences – The number of instances the particular issue is identified in the html code.
    • If the number of Occurrences is greater than zero (0), then the number will be a hyperlink. If selected, you will be brought to the Occurrences Report.
  • Pages – The number of pages that contained at least one instance of the issue.
    • If the number of pages is greater than zero (0), then the number will be a hyperlink. If selected, you will be brought to the Occurrences Report.
  • Highest Page Level – The highest level where this issue was found during the page crawl, e.g., if the Audit started on the homepage, then the homepage is Level 1. If there are links on the homepage then when we crawl to that secondary link, that would be Level 2, etc.
  • Priority Level – All Guideline tests are assigned a priority level of 1, 2, or 3. In the case of WCAG, it relates to Level A, AA, or AAA.
  • Guideline – The Guideline or test that was run against one or more pages.
    • Each Guideline listed will also be a hyperlink. When selected, a new tab will open to the WCAG technique on the W3C site that defines the criteria tested.
  • State – The result of the test performed. The result will either indicate Fail, Warning, or Visual.

Default Priority Order

Severity, Impact, Priority, Occurrences, then Issue.

See More

By default, the Issues Found list will only display the first 20 items. In order to see additional items, you will have to select “See More …” at the bottom of the list. This option will reload the page to include 20 more items.

Filter

The Filter feature in the Issue Tracker report allows for focus on the items most important to you. To filter your Issue list, simply click on the first icon above the Issue table on the far right. When you click the icon, a box will open that will provide the following filter options. These options can be accessed through the tabs at the top of the box.

To filter your Issue list, simply click on the first icon above the Issue table on the far right.

When you click the icon, a box will open that will provide the following filter options. These options can be accessed through the tabs at the top of the box.

  • Severity
    • Checkbox
    • Multiple selections allowed
    • Options = High, Medium, Low
  • Impact
    • Checkbox
    • Multiple selections allowed
    • Options = High, Medium, Low
  • Priority
    • Checkbox
    • Multiple selections allowed
    • Options = 1, 2, 3
  • Guideline
    • Keyword Typeahead dropdown
    • Single selection only
    • Options = All relevant Guidelines in the issue list
  • State
    • Checkbox
    • Multiple selections allowed
    • Options = Failed, Warning, Visual

Apply: You can make selections across all filter options/tabs and when you are finished, you must click on the Apply button in order to apply the filter to your Issue list.

Remove: To remove a single filter option, you must select that option from the specific tab and deselect the option.

Cancel: If you wish to clear all your filter options, then click on Cancel button. The box will close, and no filters will be applied.

Note: Once you have applied your filters, you can bookmark this page with the filters applied by selecting the URL in the address bar and copy/paste in a safe place.

Sort

The Sort option allows you to prioritize the Issues list based on what is most important to you.

To sort your Issue list, simply click on the second icon above the Issue table on the far right.

When you click the icon, a box will open that will provide the following filter options. This allows for a single sort option.

  • Default sort (as defined above in Issue Found section)
  • Impact (High to low)
  • Severity (High to low)
  • Priority Level, ascending (1 to 3)
  • Highest Page Level, ascending
  • Occurrences, descending
  • Guideline, alpha ascending
  • State, alpha ascending

Once you make a selection, you must select Sort to initiate the sort of your Issue list.

Cancel: If you select Cancel, then you will return to your current Issue list with no other Sort option applied.

Export

The Issue Tracker report allows for exporting all the data available on the issues listed. The Export icon can be found in the header bar at the same level as the page breadcrumbs but right justified.

The only required fields for export are Format and Name. Only when these are selected will the Export button be available.

Format options include:

  1. Comma Separated Value – CSV.
  2. JavaScript Object Notation – JSON.
  3. Microsoft Excel – XLSX.
  4. eXtensible Markup Language – XML.

If you applied any Filters to your Issue Found list then you will be provided with the option to only Export those issues.

Note: The Default selection for Export will be Export all issues.

Once you select Export, then the file will automatically download to your computer.

Cancel: If you select Cancel, then the dialog box will close and no issues will be exported.

4. My Profile

Settings

Name: A First and Last name is required for each Compliance Investigate User.

Email Address: Please be sure to use a valid Email address. This Email address will be used for any automated Emails from Compliance Investigate, including password reset requests.

If users wish to change their password but do not remember their Current Password, you will be able to Reset your Password.

5. FAQ

A collection of frequently asked questions:

Session Timeout

Why do I have to login every 30 minutes?

Compliance Investigate requires a user to log in every 30 minutes, regardless if they are actively navigating between pages. This is a security setting.
