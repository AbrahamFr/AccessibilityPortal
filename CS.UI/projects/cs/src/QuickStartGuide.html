SDK Quick Start

The Compliance Sheriff SDK consists of APIs that let you incorporate specific Compliance Sheriff features into your own in-house applications. Standard functions can be called from any front-end or UI that supports REST technology.

This SDK uses Swagger for its API documentation and testing

Not familiar with using REST APIs?

Review this primer: Rest API Tutorial

Prepare

Make sure your target environment has the following items:

  • Dedicated virtual directory on the same IIS that hosts Compliance Sheriff
  • Domain account credentials that can access the Compliance Sheriff database and shared data directory
  • Ability to use a JSON Web Token (JWT) for authentication during UI calls

Access

To access the Compliance Sheriff SDK documentation in Swagger, click below:

Swagger

Verify

To verify that the Compliance Sheriff SDK is successfully installed on your IIS, use the Demo API endpoint.

Authenticate

To authenticate with Swagger, follow these steps:

  1. Open the Compliance Sheriff Swagger site in your web browser.
  2. Use the Try It Now function for the Authentication API to output a Bearer (apiKey) token.
  3. Click Authorize and copy the token (without quotation marks) from the Response Body into the Value field, using this syntax:
    Bearer token_key_value_without_quotation_marks
  4. Click Authorize to submit your token
  5. Swagger returns the authorization and header information, and the Authorize function displays as locked.

Use

To use one of the Compliance Sheriff APIs, follow these steps:

  1. For each feature you want to implement, review the Swagger documentation for the corresponding verbs that can act on models and their properties. Required properties are marked with an asterisk(*).
  2. Ensure your authentication header includes the Bearer (apiKey) token. Authentication expires every 5 minutes.
  3. Ensure your UI call includes the appropriate verb, properties, and target endpoint.

Validate

  1. To see which roles you have been assigned, decrypt the JWT/Bearer token.
  2. To validate your API call, use the test function provided by Swagger. This function requires authentication with Swagger.
